# 串联系统冗余优化（Greedy + Backtracking）

本仓库实现了一个 10 子系统串联系统的冗余分配优化，对比传统贪心与多种改进策略，并通过回溯实现全局精修。

## 1. 问题定义

目标：在满足系统可靠度约束的前提下最小化总成本。

- 系统可靠度：
\[
R_s=\prod_{i=1}^{n}\left(1-(1-r_i)^{x_i}\right)
\]
- 总成本：
\[
C=\sum_{i=1}^{n}c_i x_i
\]
- 约束：\(R_s \ge R_{target}\), 且 \(x_i \in \mathbb{Z}, x_i \ge 1\)

---

## 2. 数据设置

- `r = [0.60, 0.85, 0.90, 0.75, 0.88, 0.92, 0.82, 0.78, 0.86, 0.89]`
- `c = [80, 10, 5, 60, 8, 3, 12, 40, 15, 7]`
- `w = [1.0, 1.0, 1.0, 1.5, 1.0, 1.0, 1.0, 1.2, 1.0, 1.0]`
- `R_target = 0.98`

---

## 3. 算法说明

### 3.1 传统贪心
每一步选择“绝对可靠度增量”最大的子系统。

### 3.2 改进1（性价比）
按 \(\Delta R_i / c_i\) 选择，优先单位成本收益高的子系统。

### 3.3 改进2（性价比 × 不稳定优先）
在改进1基础上乘以 \((1 - R_i)\)，优先补短板子系统。

### 3.4 改进3（性价比 × 不稳定 × 重要度）
在改进2基础上乘以权重 \(w_i\)，体现关键子系统优先。

### 3.5 最终集成（改进3 + 回溯）
先用改进3快速得到高质量可行解，再用回溯进行全局精修，获得更优成本。

---

## 4. 实验结果（当前版本）

- 传统贪心：`cost=1132, Rs=0.981003, steps=27`
- 改进1（性价比）：`cost=1037, Rs=0.980729, steps=30`
- 改进2（性价比×不稳定）：`cost=1082, Rs=0.980942, steps=27`
- 改进3（性价比×不稳定×重要度）：`cost=1082, Rs=0.980942, steps=27`
- 最终集成（改进3+回溯）：`cost=1033, Rs=0.980029, x*=[5,4,4,4,4,4,4,4,4,3]`

### 结果解读（简版）
1. 改进策略整体优于传统贪心（成本更低）。  
2. 纯贪心中，改进1在本组参数下成本最低。  
3. 引入回溯后达到全局更优成本（1033），体现“贪心提速 + 回溯保优”的互补性。  
4. 回溯最优解的 \(R_s\) 接近 0.98，符合“满足约束即可、成本最小优先”的优化目标。

## 7. 结论

该实现验证了两阶段框架在串联系统冗余优化中的有效性：
- 第一阶段（多维贪心）快速逼近优质可行解；
- 第二阶段（回溯）进行全局纠偏并锁定更优成本。
