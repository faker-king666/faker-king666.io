# 串联系统备件优化配置求解器 (Spare Allocation Optimizer)

🎓 **本项目为本科毕业设计核心算法模块**  
旨在解决高可靠性要求的串联系统在面临有限成本约束时，如何进行最优的冗余备件分配，以实现**“在满足目标可靠度的前提下，系统总成本最低”**。

## 💡 核心算法架构 (Algorithms)

本项目摒弃了传统穷举法在面临大规模节点时产生的“维度灾难（组合爆炸）”，创新性地采用了**“启发式贪心 + 分支定界（回溯剪枝）”**的融合架构：

1. **多维启发式贪心算法 (Heuristic Greedy Algorithm)**
   - **作用**：探路与划定边界。
   - **原理**：基于“最高性价比（可靠度增量/成本）”、“优先补偿最不稳定节点”以及“业务权重”三个维度构建综合贪心指数（Greedy Index）。
   - **特点**：在毫秒级时间内给出一个高度近似的次优解，并为后续回溯提供极其重要的**成本上限（Cost Upper Bound）**。

2. **带双重剪枝机制的回溯算法 (Backtracking with Pruning)**
   - **作用**：纠偏与寻找全局绝对最优解。
   - **剪枝策略 1（成本上限剪枝）**：利用贪心算法得出的成本作为门槛，搜索路径中一旦超支立刻回头。
   - **剪枝策略 2（可靠度下限剪枝）**：利用串联系统概率相乘递减的特性，若当前累加可靠度已低于目标值 $R_{target}$，直接剪枝。

## 🚀 快速开始 (Quick Start)

### 环境依赖
- Python 3.6+
- 无需安装任何第三方外部库（纯 Python 标准库实现）

### 运行方式
直接在终端或命令行运行核心算法脚本：
```bash
python allocation_optimizer.py
```

## 📊 经典算例演示 (Example & Output)

对于一个 4 节点的串联系统测试用例，目标可靠度要求 $R_{target} = 0.95$。

**运行结果如下：**
```text
=== 1. 执行贪心算法 ===
分配方案: x = [3, 2, 4, 2]
可靠度: Rs = 0.964574012002
总成本: Cost = 47
迭代次数: 7

=== 2. 执行回溯+剪枝 (以贪心Cost为上限) ===
全局最优方案: x* = [2, 2, 5, 2]
最高性价比可靠度: Rs* = 0.951472050883
最低总成本: Cost* = 45
回溯搜索节点数: 127 (得益于剪枝，计算量极小)

=> 结论：回溯法成功纠正了贪心算法的局部最优，节省了成本 2！
```

**结果解析：**
得益于双重剪枝机制，纯穷举原本需要上千次的无效计算，在融合算法下**仅需搜索 127 个节点**便锁定了全局唯一的最优解，完美克服了纯贪心算法由于离散变量导致的“成本超调（Overshoot）”现象.
